# API v2 — Grammar / Autocorrection (Автокоррекция)

Neural network-based autocorrection. Fixes grammar, spelling, and punctuation in one pass.

## Endpoints

- **Submit:** `POST https://textovod.com/api/v2/text/grammar`
- **Poll:** `POST https://textovod.com/api/v2/text/grammar/result`

## Authentication

Bearer token in Authorization header:

```
Authorization: Bearer ${TEXTOVOD_API2_TOKEN}
```

## Limits

- Max text length: 35,000 characters
- Rate limit: 5 requests per second

## Submit Request

**Important:** Do NOT send the `id` field — the API generates its own UUID.

```bash
JSON_BODY=$(jq -n --arg text "$TEXT" '{text: $text, lang: "ru"}')

curl -s -X POST https://textovod.com/api/v2/text/grammar \
  -H "Authorization: Bearer ${TEXTOVOD_API2_TOKEN}" \
  -H "Content-Type: application/json" \
  -d "$JSON_BODY"
```

### Submit Response

```json
{
  "status": 3,
  "id": "edf491d8-9e70-4594-8b05-c265184963d8"
}
```

- `status: 3` — task queued
- `id` — UUID generated by the API, use for polling

## Poll Request

```bash
POLL_BODY=$(jq -n --arg id "$TASK_ID" '{id: $id}')

curl -s -X POST https://textovod.com/api/v2/text/grammar/result \
  -H "Authorization: Bearer ${TEXTOVOD_API2_TOKEN}" \
  -H "Content-Type: application/json" \
  -d "$POLL_BODY"
```

### Poll Response — Processing

```json
{
  "status": 1
}
```

### Poll Response — Ready

The response contains a `messages` array with both the original (`user`) and corrected (`assistant`) text:

```json
{
  "status": 2,
  "messages": [
    {
      "role": "user",
      "content": "Мы прийдём завтра на встречу чтобы обсудить новые предложения"
    },
    {
      "role": "assistant",
      "content": "Мы приедем завтра на встречу, чтобы обсудить новые предложения."
    }
  ]
}
```

## Status Codes

| Code | Meaning |
|------|---------|
| 1 | Processing |
| 2 | Ready (result available) |
| 3 | Queued |
| 30 | Insufficient balance |

## Parsing Results

```bash
# Get corrected text (assistant message)
echo "$RESULT" | jq -r '.messages[] | select(.role == "assistant") | .content'

# Get original text (user message)
echo "$RESULT" | jq -r '.messages[] | select(.role == "user") | .content'

# Check status
echo "$RESULT" | jq '.status'
```

## Poll Strategy

- Wait 3 seconds between polls
- Max 15 attempts
- Total max wait: ~45 seconds
- Make each poll as a separate Bash tool call

## Error Cases

- Missing or invalid token — `{"status": 0, "messages": ["..."]}` with error text
- Sending `id` in request body — `{"status": 0, "messages": ["Ошибка формата JSON"]}`
- Text exceeds 35K chars — error before processing
- Empty text — error
- Status 30 — insufficient balance, prompt user to top up at textovod.com
